# Phase 4 Plan: WF6 Report Sender

## Objective

Deploy WF6 Report Sender -- a workflow that receives the finished HTML report + PDF binary from WF5 (via the Master Orchestrator) or loads the latest report data from Supabase (standalone mode), sends a professional HTML email with the PDF as attachment via Gmail, logs the send in Supabase `workflow_runs`, and returns a structured response. The workflow must be triggerable both standalone (via its own Webhook) and from the Master Orchestrator (Phase 5).

---

## WF6: Report Sender

### Overview

**Trigger**: Webhook (POST) -- accepts calls from the Master Orchestrator or standalone invocation.
**Purpose**: Send the SocialPulse weekly report via Gmail. HTML body for direct reading, PDF attachment for archiving/sharing.
**Error Handling**: Gmail send uses retry. Run log update uses `onError: continueRegularOutput`. Structured `{ success, data, errors }` response.

### High-Level Flow

```
Webhook Trigger (POST)
  |
  v
IF: Dual-Trigger Pruefung (report data in body?)
  |
  +-- TRUE (Master call) --> Report-Daten zusammenfuehren
  |                          (htmlReport + PDF binary + config from Master)
  |
  +-- FALSE (standalone) --> Google Sheets: Konfig lesen
                               |
                               v
                            Code: Standalone-Daten vorbereiten
                            (build minimal report or read from Supabase)
                               |
                               v
                            Report-Daten zusammenfuehren
  |
  v
Code: E-Mail vorbereiten (Subject, To, CC, HTML body, attachment filename)
  |
  v
Gmail: Report versenden (HTML body + PDF attachment)
  |
  v
HTTP Request: Supabase Run-Log schreiben
  |
  v
Code: Response zusammenbauen
  |
  v
Respond to Webhook
```

### Architecture Decision: Direct Gmail Node

The native `nodes-base.gmail` node with `operation: "send"` (typeVersion 2.1) fully supports:
- HTML email body
- File attachments from binary data
- Multiple recipients (comma-separated)
- CC recipients

This is the correct choice. No HTTP Request needed for sending email.

### Architecture Decision: PDF Binary Handling

When called from the Master, WF6 receives the PDF binary data that was generated by WF5's HTML-to-PDF node. n8n passes binary data between workflows via the Execute Workflow node. The webhook POST body carries the JSON report data, and the binary attachment is passed through n8n's internal binary data handling.

For standalone mode (testing), WF6 does not have the PDF. It sends a text-only email explaining that the full report is available via the Master pipeline.

#### Node 7: Report versenden

| Property | Value |
|---|---|
| **ID** | `wf6-gmail-send` |
| **Name** | `Report versenden` |
| **Type** | `n8n-nodes-base.gmail` |
| **typeVersion** | `2.1` |
| **Position** | `[1300, 460]` |
| **Credentials** | Gmail OAuth2 |
| **retryOnFail** | `true` |
| **maxTries** | `3` |
| **waitBetweenTries** | `5000` |

**Parameters:**
```json
{
  "sendTo": "={{ $json.sendTo }}",
  "subject": "={{ $json.subject }}",
  "emailType": "html",
  "message": "={{ $json.emailHtml }}",
  "options": {
    "ccList": "={{ $json.cc }}",
    "attachmentsUi": {
      "attachmentsBinary": [
        {
          "property": "data"
        }
      ]
    }
  }
}
```

**Notes:**
- `emailType: "html"` -- sends the full HTML report as the email body.
- `sendTo` accepts comma-separated email addresses.
- `ccList` in options for CC recipients.
- `attachmentsBinary` references the binary property `data` which contains the PDF from WF5. If no binary data exists (standalone mode), Gmail sends without attachment.
- Gmail typeVersion 2.1 supports the `options.attachmentsUi` parameter for binary attachments.
- Retry 3x/5s for Gmail API reliability.

**Gmail Send Parameters (typeVersion 2.1):**
- `sendTo` (string): comma-separated recipient emails
- `subject` (string): email subject
- `emailType` (options): "html" or "text"
- `message` (string): email body (HTML when emailType is "html")
- `options.ccList` (string): comma-separated CC emails
- `options.attachmentsUi.attachmentsBinary` (array): binary properties to attach
